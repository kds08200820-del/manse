# 3·1만세운동본부 홈페이지 — 설치 안내 (처음부터 도메인까지)

이 폴더(`agent/manse`)는 **기존 연합회 홈페이지와 똑같은 구조**로 만든 새 사이트 시작 템플릿입니다.
로그인·회원관리·게시판(갤러리)·공지·자료실·언론 등 기능이 그대로 들어 있고,
**설정 3줄과 내용 일부만 바꾸면** 3·1만세운동본부 홈페이지가 됩니다.

> 이미 기존 사이트에서 해보신 과정과 **완전히 동일**합니다. 새 계정/새 저장소/새 DB만 따로 만든다고 생각하시면 됩니다.

---

## 0. 구조 한눈에
- **화면**: 순수 HTML/CSS/JS (정적 사이트) → **GitHub Pages**로 무료 배포
- **회원/데이터**: **Supabase**(로그인·회원·게시판·공지 등)
- **사진·파일 저장**: **Cloudflare Worker + R2**(업로드)
- **주소**: 내 **도메인**을 GitHub Pages에 연결

즉 기존 사이트와 동일합니다. 기존 사이트 것과 **섞이지 않도록** Supabase 프로젝트·Cloudflare Worker·GitHub 저장소·도메인을 **새로 하나씩** 만듭니다.

## 준비물
- GitHub 계정, Supabase 계정, Cloudflare 계정 (기존 사이트 때 쓰던 계정 그대로 OK)
- 도메인 1개 (아래 6단계에서 등록 안내)

## 3단계 — GitHub 새 저장소 + Pages
1. GitHub에서 **새 저장소(New repository)** 생성 (예: `manse` 또는 `31manse`). Public 권장.
2. 이 폴더를 그 저장소에 올립니다 (터미널):
   ```
   cd "C:\Users\kds08\OneDrive\바탕 화면\agent\manse"
   git init
   git add -A
   git commit -m "3·1만세운동본부 홈페이지 시작"
   git branch -M main
   git remote add origin https://github.com/<내계정>/<저장소>.git
   git push -u origin main
   ```
   (제가 대신 push까지 진행해 드릴 수도 있습니다. 저장소를 만들고 주소만 주세요.)
3. GitHub 저장소 → **Settings → Pages** → Source: `Deploy from a branch`, Branch: `main` / `/(root)` → Save.
4. 몇 분 뒤 `https://<내계정>.github.io/<저장소>/` 로 뜹니다. (도메인은 6단계에서 연결)

> ⚠️ `CNAME` 파일에 지금 `YOUR-DOMAIN.kr` 로 되어 있습니다. 도메인 정하면 그 값으로 바꾸세요(6단계).

## 4단계 — Supabase 새 프로젝트 + DB
1. Supabase → **New project** 생성 (이름 예: `manse`). 비밀번호·리전(Northeast Asia) 설정.
2. 생성되면 **Project Settings → API** 에서 2가지 복사:
   - **Project URL** (예: `https://xxxx.supabase.co`)
   - **anon public** 키
3. 이 값을 로컬 `supabase-config.js` 의 (1)(2)에 붙여넣기.
4. **SQL Editor** 에서 아래 순서로 실행(붙여넣고 Run):
   - **먼저** `setup-all.sql` (회원·갤러리·관리자·임원방 등 기본 전부)
   - **그다음** 나머지 기능 SQL을 하나씩(재실행 안전):
     `dashboard-setup.sql`, `history-setup.sql`, `press-setup.sql`, `hero-setup.sql`,
     `officer-photos-setup.sql`, `archive-docs-setup.sql`, `archive-hidden-setup.sql`,
     `archive-titles-setup.sql`, `member-admin-setup.sql`, `roster-roles-setup.sql`,
     `gallery-board-setup.sql`
5. **Authentication → URL Configuration**:
   - **Site URL**: `https://YOUR-DOMAIN.kr` (도메인 정한 뒤)
   - **Redirect URLs** 에 추가: `https://YOUR-DOMAIN.kr/reset-password.html` (비밀번호 재설정용)
   - **Email → Confirm email** 은 **끄기(OFF)** (가입 즉시 로그인 방식)

## 5단계 — Cloudflare Worker + R2 (사진·파일 업로드)
1. Cloudflare → **R2** 에서 새 **버킷** 생성 (예: `manse-files`).
2. **Workers & Pages** → 새 Worker 생성 (예: `manse-files`). 코드 편집기에 이 폴더의
   `cloudflare-worker/gallery-worker.js` **전체를 붙여넣고 배포**.
3. Worker **Settings → Variables and Secrets** 에 등록:
   - `SUPABASE_URL` = 4단계의 Project URL
   - `SUPABASE_ANON_KEY` = anon 키
   - `ADMIN_EMAIL` = 최고관리자 이메일(예: `kds08200820@gmail.com`)
   - `SUPABASE_SERVICE_ROLE_KEY` = Supabase → Settings → API → **service_role** (비밀, 외부노출 금지) — 관리자 임시비밀번호 기능용
   - Worker에서 R2 버킷을 **Bindings**로 연결: 변수명 `BUCKET` → 위 R2 버킷
4. 배포되면 Worker 주소(`https://manse-files.<계정>.workers.dev`)를 `supabase-config.js` 의 (3) `R2_WORKER_URL` 에 붙여넣기.

## 6단계 — 도메인 등록 + 연결
1. **도메인 등록**: 가비아(gabia.com)·후이즈·Cloudflare Registrar 등에서 원하는 주소 구입
   (예: `manse31.kr`, `hs31manse.or.kr` 등). `.kr`/`.or.kr`/`.com` 아무거나.
2. **DNS 설정**(도메인 산 곳의 DNS 관리에서):
   - 루트(`@`) → A 레코드 4개(GitHub Pages IP): `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - `www` → CNAME → `<내계정>.github.io`
   - (Cloudflare로 DNS 관리 시 같은 값, 프록시는 회색 구름/DNS only 권장)
3. 로컬 `CNAME` 파일 내용을 산 도메인으로 바꾸고 push (예: `manse31.kr`).
4. GitHub → Settings → **Pages → Custom domain** 에 그 도메인 입력 → **Enforce HTTPS** 체크(인증서 발급까지 몇 분~1시간).
5. Supabase(4-5단계)와 `sitemap.xml`/`robots.txt` 의 `YOUR-DOMAIN.kr` 도 실제 도메인으로 교체.

## 7단계 — 관리자 지정 & 첫 로그인
- 홈페이지에서 **회원가입** → 그 계정이 `ADMIN_EMAIL`(위 5-3) 과 같으면 자동으로 **최고관리자**가 됩니다.

## 막히면
기존 사이트에서 겪은 것과 **같은 문제/해결**이 그대로 적용됩니다.
- 배포가 안 보이면: GitHub에 빈 커밋 하나 push(재빌드 트리거) 또는 몇 분 대기, `Ctrl+F5`.
- “테이블 없음/컬럼 없음” 오류: 해당 `*-setup.sql` 을 SQL Editor에서 실행.
- 업로드 실패: Worker 재배포 + R2 바인딩/시크릿 확인.
